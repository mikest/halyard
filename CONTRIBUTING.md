# Contributing to Halyard

Thanks for your interest in contributing! Please read this before opening a PR.


## Before You Start
This library is primarily developed for use in my game project. Compatibility is not guaranteed between releases.
If you're unsure whether your change is in scope, open an issue and discuss it with me first to avoid wasted effort. We'll make a plan first.


## Reporting Bugs
Bug reports are always welcome. Please include:
- Godot version and platform
- A minimal reproduction scene or script
- What you expected to happen vs. what actually happened


## Pull Requests
A few hard rules:

- **Explain your code.** If you can't describe what your change does and why, the PR will be rejected.
- **No slop code.** If your PR looks like slop, it will be rejected. I may also block you.
- **No drive-by cleanups.** Reformatting, renaming, or reorganizing code that you did not author.


### Code Style
Follow the conventions in the existing code. If I finally got around to using a format description, please use it.


### Scope
Good candidates for PRs:
- Bug fixes with a clear reproduction case
- Platform-specific build or compatibility fixes
- Performance improvements with measurable benchmarks

Out of scope:
- New features not discussed in an issue first
